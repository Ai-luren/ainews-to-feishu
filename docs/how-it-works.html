系统架构

一张图看懂整个系统

四个角色，三步传递，每天自动完成。

为什么不直接用 GitHub 的定时任务？ GitHub Actions 自带的 cron 有约 1 小时随机延迟，早报会拖到中午。cron-job.org 每 15 分钟主动 "敲门"，橘鸦发布后 15 分钟内你就能收到。
每日运行流程

每次触发，7 个步骤自动完成

全程无需人工操作，出错了会自动发告警。

步骤 1

cron-job.org 定时触发

北京时间 08:00–12:00，每 15 分钟 向 GitHub 发一次"请运行"请求。

⏰ 全自动
步骤 2

GitHub Actions 启动

云端服务器启动，安装依赖，开始跑 push.py 推送脚本。约 10 秒完成。

☁️ 云端运行
步骤 3

检查今天推过了吗？

读取 state.json 里记录的最后推送日期。如果已是今天，直接结束，不重复推。

↩ 已推过 → 跳过
步骤 4

抓取橘鸦 RSS

从 RSS 源里查找今天（北京时间）的内容。没找到说明橘鸦还没发，等下一个 15 分钟。

⏳ 未更新 → 等待
步骤 5

解析成飞书富文本卡片

把 HTML 内容按分类（要闻 / 模型发布 / 开发生态…）解析，自动排版成飞书卡片格式。

🃏 自动排版
步骤 6

推送卡片到飞书群

通过群机器人 Webhook 发送。若连续失败 3 次，自动发告警到同一个群提醒你处理。

✅ 推送成功
步骤 7 · 收尾

更新 state.json，防止今天重复推

把今天日期写入 state.json 并提交到 GitHub。之后今天剩余所有触发都会在步骤 3 自动跳过，每天飞书群只会收到一张卡片。

[ok] pushed 成功推出
[skip] already pushed today 今天已推
[skip] juya not updated yet 源头未发布
[fail] 推送失败
关键配置

这 4 件事可以随时改

改配置不需要动代码，一条命令就能生效。

换飞书群

在新群建机器人后，更新这 4 个 Secret。旧群机器人记得手动删除防泄露。

gh secret set LARK_WEBHOOK_URL gh secret set LARK_WEBHOOK_SECRET gh secret set LARK_OPS_WEBHOOK_URL gh secret set LARK_OPS_WEBHOOK_SECRET

换 RSS 源

存在 GitHub Variable 里，不设置默认用橘鸦。换源不需要改任何代码。

gh variable set RSS_URL \ --body "https://新的RSS地址" \ -R /design-team-ai-daily

改推送时间

直接去 cron-job.org 网页改时间窗口，不需要改代码，改完立即生效。

# 在 cron-job.org 网站修改 # 默认：08:00–12:00（北京时间） # Crontab: */15 8-12 * * * # 时区: Asia/Shanghai

暂停 / 恢复

临时不想推了，一条命令停止，想恢复直接 enable，配置全保留不丢失。

# 暂停（次日起不再推送） gh workflow disable daily-ai-news.yml # 恢复 gh workflow enable daily-ai-news.yml
换新电脑 / 换公司

5 步重新激活，10 分钟搞定

推送逻辑跑在 GitHub 云端，和本地电脑完全无关。换电脑只需恢复「操控工具」。

1

安装 GitHub CLI 并登录

新电脑没有的话先装。它是操控整个系统的核心工具。

brew install gh && gh auth login
2

克隆仓库到本地

把仓库拉到本地，里面有所有脚本和 Skill 文件，也是你的运维中心。

git clone https://github.com//design-team-ai-daily.git ~/design-team-ai-daily
3

激活 Claude Code Skill（可选但推荐）

激活后可以直接跟 Claude 说「换飞书群」「排障」，它会自动一步步引导你操作，不用记命令。

mkdir -p ~/.claude/skills && ln -s ~/design-team-ai-daily/skills/daily-ai-news ~/.claude/skills/daily-ai-news
4

换了新飞书群？更新 4 个 Webhook Secret

在新群建机器人后依次运行，终端提示粘贴时直接粘贴 URL 和 Secret（不会回显，安全）。

gh secret set LARK_WEBHOOK_URL -R /design-team-ai-daily gh secret set LARK_WEBHOOK_SECRET -R /design-team-ai-daily gh secret set LARK_OPS_WEBHOOK_URL -R /design-team-ai-daily gh secret set LARK_OPS_WEBHOOK_SECRET -R /design-team-ai-daily
5

手动触发一次，确认一切正常

看飞书群是否收到卡片。日志显示 [ok] pushed 或 [skip] 都是正常状态。

gh workflow run daily-ai-news.yml -R /design-team-ai-daily gh run watch -R /design-team-ai-daily
